## Index Strategy

### Primary Indexes
- All tables have UUID primary keys

### Composite Indexes
- `idx_notifications_user_read` - User + Read status for notification queries
- `idx_student_progress_course` - Course progress tracking
- `idx_payroll_employee_month` - Payroll period queries
- `idx_marketing_metrics_platform_date` - Time-series analytics

### Single Column Indexes
- Foreign key columns for join performance
- `created_at` columns for time-based queries
- Status columns for filtering

## RLS Coverage

All tables have RLS enabled with appropriate policies:
- User-specific tables: Owner-only access
- Admin tables: Admin role access
- Public tables: Read access to all, write to authenticated
- System tables: Admin/Technical roles only
